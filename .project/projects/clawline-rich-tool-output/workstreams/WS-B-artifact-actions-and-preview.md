# Workstream: WS-B Artifact Actions and Preview

## Objective

Improve how generated artifacts and action-heavy tool results can be previewed, opened, copied, shared, or retried on mobile.

## Owned Files/Areas

- artifact cards and preview affordances
- session-safe follow-up actions such as open, copy, share, and retry
- artifact/session coherence across transcript and dedicated views

## Dependencies

- WS-A output classification identifies the first artifact-rich flows to support
- existing artifact surfaces can be extended safely

## Risks

- action affordances may differ by output type and platform
- overlap with multimedia sharing could create duplicated logic

## Handoff Criteria

- first supported artifact/result types offer useful follow-up actions
- preview and action affordances feel native on mobile
- transcript and artifact views feel consistent
