# Workstream: WS-D Quality and Alpha Readiness

## Objective
Create enough test discipline, smoke evidence, and rollout structure to make the first alpha trustworthy.

## Owned Files/Areas
- Unit and widget test setup.
- Smoke test notes and route checklist.
- Alpha rollout and readiness artifacts.
- Hardening tasks for connection, empty, error, and retry states.

## Dependencies
- Depends on meaningful progress in WS-A, WS-B, and WS-C.

## Risks
- Calling the app alpha-ready based only on happy-path demos.
- Under-testing reconnect, empty state, and degraded backend behavior.

## Handoff Criteria
- Core routes have smoke evidence.
- Major alpha blockers are documented.
- The repo has a clear readiness checklist for Bart’s internal usage.
