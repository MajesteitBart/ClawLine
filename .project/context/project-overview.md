# Project Overview

## Mission
- Build ClawLine as a Flutter-based OpenClaw mobile product, Android first.
- Turn the architecture work into an execution-ready Delano portfolio that can ship a real alpha, then continue cleanly into power-user and advanced operator capabilities.
- Make mobile OpenClaw use materially better than Telegram for daily operator workflows while preserving a credible path to deeper console and node capabilities.

## Active Delivery Scopes
- `clawline-android-alpha`: internal alpha for gateway connection, session-first chat, activity visibility, and read-first console value.
- `clawline-android-power-user`: phase 2 work for organization, approvals, skills, cron, device clarity, and richer voice/composer ergonomics.
- `clawline-android-advanced-platform`: later-stage platform work for advanced session flows, deeper admin surfaces, artifacts, offline behavior, and collaborator-grade operation.

## Current Health
- Delano runtime is installed and the Flutter skill pack is available in `.agents/skills/`.
- The architecture baseline is captured in `docs/architecture_outline.md` and now maps to a three-project Delano portfolio instead of a single alpha bucket.
- The repository is planning-ready and validation-clean, but implementation has not started yet.
- Main near-term risks are scope creep, gateway contract drift, and losing momentum by over-planning instead of moving into Milestone 1 execution.
