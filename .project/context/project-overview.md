# Project Overview

## Mission
- Build ClawLine as a Flutter-based OpenClaw mobile product, Android first.
- Turn the architecture work into an execution-ready Delano repo that can ship an internal alpha for Bart.
- Make mobile OpenClaw use materially better than Telegram for daily operator workflows.

## Active Delivery Scopes
- `clawline-android-alpha`: first alpha project for the mobile app shell, gateway connectivity, session-first chat foundation, and operator console surfaces.

## Current Health
- Delano runtime is installed and the Flutter skill pack is available in `.agents/skills/`.
- The architecture baseline is captured in `docs/architecture_outline.md` and is strong enough to guide implementation.
- No Flutter application code exists yet, so the current repository state is planning- and scaffold-heavy rather than product-ready.
- Main near-term risks are scope creep, gateway contract drift, and overbuilding admin depth before the session-first mobile core is solid.
